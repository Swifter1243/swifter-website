Excel Software Renderer is an experimental renderer which draws to "pixels" in an excel sheet.

For an assignment in our python course at school, we were tasked with writing some program that utilizes any three python packages. I decided this was an opportunity to showcase my knowledge of the graphics pipeline, and utilized openpyxl, pywavefront, and numpy to create a software renderer.

The model inside of Blender.

The final render inside of Excel.

The program would load an obj model using pywavefront, render it, and draw it to "pixels" (square cells) in an output Excel sheet.

The program executes the following steps:

  1. Initialize destination and depth buffers.
  2. Load the obj file.
  3. Run "vertex shader" on every vertex.
  4. Project vertices to screen using Orthographic Projection.
  5. Connect triangle edges using Bresenham's Algorithm.
  6. Interpolate edges using Scanline Rendering to produce fragments.
  7. Discard fragments which are behind objects based on the depth buffer.
  8. Run remaining fragments through "fragment shader".
  9. Write fragments to the depth and destination buffers.
  10. Finalize destination buffer to excel sheet.

The program isn't really performant or practical, but it was a fun experiment to learn how to render objects from beginning to end. In the future I could improve this program with things like perspective projection, triangle clipping, frustum culling, and more.
